# pi-bridge

HTTP + WebSocket bridge server that exposes [pi](https://github.com/mariozechner/pi-mono)'s `--mode rpc` interface over the network. Built for mobile and desktop clients that can't spawn a local CLI process.

## Architecture

```
iOS / Mac app  ──HTTP + WS──▶  pi-bridge  ──stdin/stdout──▶  pi --mode rpc
                                  │
                                  ├── APNs push ──▶  iOS device (optional)
```

The bridge manages a pool of `pi --mode rpc` subprocesses, one per session. Clients interact through a REST API for commands and a WebSocket for real-time event streaming.

## Features

- **Session management** - create, resume, list, and destroy sessions
- **Real-time streaming** - WebSocket event stream with reconnect support (`?sinceSeq=N`)
- **Turn tracking** - monotonic `seq` and `turnId` on every event for dedup and stale-event rejection
- **Ring buffer replay** - late-joining WebSocket clients receive recent events, with `resync_required` sentinel on buffer overflow
- **Multi-client fanout** - multiple devices can subscribe to the same session
- **APNs push notifications** - optional, fires on `agent_end` with smart filtering (skips foregrounded devices viewing that session)
- **Image attachments** - forward base64 images from client to the RPC process
- **Model switching** - change provider/model per session at runtime
- **Thinking level** - adjust reasoning effort (off, minimal, low, medium, high, xhigh)
- **Context compaction** - trigger manual compaction with optional custom instructions
- **Keep-alive** - opt individual sessions out of the 30-min idle auto-reaper
- **Bearer token auth** - on all requests (header or `?token=` query param for WebSocket)
- **Sandboxed Docker mode** - containerized test bridge with model locking for App Store review

## Requirements

- [Bun](https://bun.sh) runtime
- [pi CLI](https://github.com/mariozechner/pi-mono) installed and available in PATH

## Quick start

```bash
# 1. Clone and enter the directory
git clone https://github.com/user/pi-bridge.git
cd pi-bridge

# 2. Install dependencies
bun install

# 3. Start the server (auto-generates a token on first run)
./run.sh
```

`run.sh` stores a token at `~/.config/pi-bridge/token` (created automatically on first run) and starts the server on port 7171.

## Configuration

All configuration is via environment variables:

| Variable | Required | Default | Description |
|---|---|---|---|
| `PI_BRIDGE_TOKEN` | Yes | (auto-generated by `run.sh`) | Bearer token for auth |
| `PI_CLI` | No | `pi` | Path to the pi CLI binary |
| `PI_BRIDGE_PORT` | No | `7171` | Port to bind |
| `PI_BRIDGE_HOST` | No | `0.0.0.0` | Host to bind |

### Optional: APNs push notifications

Store these in `~/.config/pi-bridge/apns.env` (sourced by `run.sh`):

| Variable | Description |
|---|---|
| `APNS_KEY_PATH` | Path to APNs Auth Key `.p8` file |
| `APNS_KEY_ID` | 10-char Key ID from Apple Developer portal |
| `APNS_TEAM_ID` | 10-char Team ID |
| `APNS_BUNDLE_ID` | App bundle identifier (used as `apns-topic`) |
| `APNS_ENV` | `sandbox` or `production` (default: `sandbox`) |

### Optional: model locking (test/sandbox mode)

| Variable | Description |
|---|---|
| `PI_FORCE_PROVIDER` | Lock all sessions to this provider |
| `PI_FORCE_MODEL` | Lock all sessions to this model |

## API

### Health

```
GET /health
```

Public endpoint (no auth required). Returns bridge status.

### Sessions

```
GET    /sessions                          # List on-disk + live sessions
POST   /sessions                          # Create (or resume) a session
DELETE /sessions/:id                      # Destroy a live session
GET    /sessions/:id/messages             # Get message history
GET    /sessions/:id/state                # Get session state
GET    /sessions/:id/models               # List available models
POST   /sessions/:id/prompt              # Send a user message
POST   /sessions/:id/steer               # Steer the agent mid-turn
POST   /sessions/:id/follow_up           # Follow-up message
POST   /sessions/:id/abort               # Abort current generation
POST   /sessions/:id/model               # Switch model
POST   /sessions/:id/thinking            # Set thinking level
POST   /sessions/:id/compact             # Trigger context compaction
POST   /sessions/:id/keep-alive          # Toggle idle-reaper opt-out
POST   /sessions/:id/name                # Rename session
```

### WebSocket

```
GET /sessions/:id/events?token=...&sinceSeq=N
```

Upgrade to WebSocket for real-time event streaming. Supports `sinceSeq` for reconnect dedup.

### Push notifications (iOS)

```
POST   /sessions/:id/notifications/subscribe    # Register APNs device token
DELETE /sessions/:id/notifications/subscribe    # Unregister device token
POST   /devices/:token/active                   # Set device foreground state
POST   /devices/:token/viewing                  # Set which session device is viewing
```

## Docker (sandboxed test bridge)

For App Store review or isolated testing:

```bash
# 1. Copy and configure env
cp .env.test.example .env.test
# Edit .env.test with your values

# 2. Build and run
./run-test-container.sh
```

This runs the bridge in a locked-down container with no host filesystem access.

## Source files

| File | Description |
|---|---|
| `index.ts` | HTTP + WebSocket server, routing, push notification logic |
| `manager.ts` | Session pool, event fanout, ring buffer, seq/turnId stamping, idle reaper |
| `rpc.ts` | Spawns `pi --mode rpc`, newline-delimited JSON over stdin/stdout |
| `sessions.ts` | Scans `~/.pi/agent/sessions/` for on-disk session files |
| `apns.ts` | Apple Push Notification service client (HTTP/2, token-based .p8 auth) |
| `types.ts` | TypeScript types for the pi RPC protocol |
